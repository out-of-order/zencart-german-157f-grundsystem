Zen Cart 1.5.7g deutsch 

Version vom 16.11.2023

#####################################################################################################################################

1.5.7g deutsch ist ein sehr umfangreicher Service Pack für 1.5.7 deutsch
Es integriert die fast vollständige Codebasis der aktuellen amerikanischen Version 1.5.8 in die deutsche Zen Cart Version
Damit wird PHP 8.1.x vollständig unterstützt.
Die integrierten Module und 3rd Party Komponenten werden auf den Stand von November 2023 gebracht.

Die Umstellung der Sprachdateien, die die amerikanische Version 1.5.8 bereits mitbringt, wird in einer für Herbst 2024 geplanten deutschen Zen Cart Version 1.5.8 mit vollständiger PHP 8.2 Unterstützung nachgezogen.
Dadurch kann in 1.5.7g weiterhin mit den gewohnten Sprachdefinitionen gearbeitet werden. 

1.5.7g behebt auch alle Fehler die seit Erscheinen von 1.5.7f aug Github gemeldet wurden.

Aktualisierte Komponenten:
- PHP Mailer Update auf 6.8.1
- MobileDetect Update auf 3.74.3
- CKEditor Update auf 4.22.1
- TinyMCE Editor Update auf 6.7.2
- braintree_web Update auf 3.97.2, braintree_php Update auf 6.14.0 
- FontAwesome Update auf 6.4.0
- jQuery Update auf 3.7.0
- Image Handler Update auf 5.3.4 mit webp Unterstützung

Neue Funktionalitäten:
- Kunden können Kundengruppen zugeordnet werden. Plugins könnten bestimmte Funktionen nur für bestimmte Kundengruppen bereitstellen.
- Unterstützung des Plugin Managers für Module, die bereits dieses Format nutzen

Verbesserungen:
- Vollständige Unterstützung von PHP 8.1
- Neustrukturierung wichtiger Funktionsdateien
- Performanceoptimierung durch Zentralisierung von Klassen und Funktionen 
- Admin Layout durchgehend augenfreundlich und mobilefreundlich


Wer bereits 1.5.7, 1.5.7e oder 1.5.7f verwendet kann mit dem speziellen Updatepaket in UPDATE-VON-157-AUF-157g auf 1.5.7g aktualisieren.
Anleitung dazu in der liesmich.txt im Ordner UPDATE-VON-157-AUF-157g

Hinweise zu Error Logs unter PHP 8:
Aufgrund des Override Systems für Sprachdateien in Zen Cart 1.5.x ist es unvermeidlich, dass es "doppelte" Sprachdefinitionen gibt.
PHP 8 wird doppelte Sprachdefinitionen mit einem Logfile quittieren und darauf hinweisen.
Unter Konfiguration > Protokollierung/Logfiles ist daher beim Logfiles Level für Adminbereich und Frontend "IgnoreDups" voreingestellt. 
Das sollte auch so bleiben, die Einstellung Yes führt zu unzähligen, völlig unnötigen Warning Logs
